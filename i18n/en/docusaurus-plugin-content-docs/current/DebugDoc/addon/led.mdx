# Extending Control to Drive LED with Plugins

Original Document

```cfg
https://github.com/julianschill/klipper-led_effect
```

## Installation

Please connect to your host machine via **WIFI** using an **SSH tool** such as **MobaXterm_Personal**, and ensure the following:

1. **Ensure that the Klipper firmware is installed on your host machine.**
2. **Make sure the user logged in is the one who has installed Klipper.**
3. **Ensure your input method is set to English.**
4. **Ensure your host machine can normally search for the device.**
5. **Ensure all the above precautions are followed; otherwise, you cannot proceed to the next step.**

This module can be installed into an existing Klipper installation using an installation script.

```cfg
cd ~
git clone https://github.com/julianschill/klipper-led_effect.git
cd klipper-led_effect
./install-led_effect.sh
```

## Reference Configuration

```cfg
[neopixel TP_led]
pin: PB7
chain_count: 25
# Number of LEDs
# Number of LEDs
color_order: GRB
initial_RED: 0.4    
initial_GREEN: 0.8
initial_BLUE: 1
initial_WHITE: 0.0
#66CCFF 

[led_effect sb_nozzle_cooling]
autostart:              false
frame_rate:             24
leds:
    neopixel:TP_led (9,10)
layers:
        breathing  3 1 top (0.0, 0.0, 1.0, 0.1)

[led_effect rainbow]
leds:
    neopixel:TP_led
autostart:                          true
frame_rate:                         24
layers:
    gradient  0.3  1 add (0.3, 0.0, 0.0),(0.0, 0.3, 0.0),(0.0, 0.0, 0.3)
```
