# RS232 Firmware Compilation

## Start Compilation

    <SSH />

    <MakeOperate/>

    <RS232 />

    * Select GPIO pins to set at micro-controller startup, input: !gpio5 Please note ! is uppercase
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/rs232.png').default} size="100%" align="center" />
    * Press `Q` key, you will see **Save configuration**, then press `Y` key
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * Now the configuration should be saved and you should be back at the command line interface
    * Input `make -j4` to start compilation, it takes a while
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * The following output indicates successful compilation

        ```bash
        Linking out/klipper.elf
        Creating uf2 file out/klipper.uf2
        ```

## Enter Burn Mode

    :::tip

    Left is the `RESET button`, right is the `BOOT button`

    :::

    * There are two ways to enter DFU
    * First method: Under completely powered-off condition of the mainboard, hold the BOOT button and connect the data transfer cable to the mainboard and the host machine
    * Second method: Connect the data transfer cable to the mainboard and the host machine, hold the BOOT button and press the RST button, then release the RST button and finally release the BOOT button
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/boot.png').default} size="100%" align="center" />
