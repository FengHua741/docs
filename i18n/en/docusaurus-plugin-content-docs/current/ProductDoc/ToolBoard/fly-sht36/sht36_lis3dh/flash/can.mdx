# CAN Firmware Compilation

## Start Compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Select GPIO pins to set at micro-controller startup, input: !GPIO20 Please note ! is uppercase
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/can.png').default} size="100%" align="center" />
    * Press `Q` key, the **Save configuration** prompt appears, then press `Y` key
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * Now you should have saved the configuration and exited to the command line interface
    * Input `make -j4` to start compilation, it takes a bit of time
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * The following output indicates successful compilation

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Burn Firmware

    * Before burning the firmware, use an XT30 cable to connect the tool board to UTOC or a mainboard with bridge firmware installed. Below is a reference wiring diagram
    * Please ensure that the DIP switches are set to the corresponding positions
        <ImageView image={require('@site/docs/General/get-id/img/utoc.png').default} size="80%" align="left" />
    * Ensure that the CAN H and CAN L are measured with a multimeter in the open circuit state and the resistance value is around `60立`. If it is `120立`, you need to connect a `120立` jumper. If it is around `40立`, you need to remove the jumper
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/120.png').default} size="50%" align="center" />
